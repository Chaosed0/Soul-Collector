Git

Setting Up

Run through this guide until you get to step 4. Do the first command, then go to your Assembla profile. Go to "Manage SSH Keys", and do a paste into the box (the command executed will have copied the ssh key, so no need to copy anything).

Using Git

First thing you want to do is to identify yourself. Run the following commands with your own info substituted:

git config --global user.email "EMAIL@EMAIL.COM"
git config --global user.name "USERNAME"

Now, to clone the repository (if you don't have a local copy):

git clone git@git.assembla.com:cmpsc483w-un.git

If you're starting work from somewhere you already have the repository, make sure to pull the most recent changes before starting work:

git pull

After making some changes, commit to your local repository (make a local backup):

git add * git commit

The second command will bring up a vim prompt to type in a commit message. Pay attention to the message at the bottom - it will tell you what changes you made. Commit messages should take the following form:

Simple, short explanation

A bit of a longer, detailed explanation here, if needed. Can span multiple lines.

To push to the remote server (our Assembla repository):

git push

If someone's made changes before you, you'll have to pull their changes first and possibly merge their changes before commiting and pushing again.
